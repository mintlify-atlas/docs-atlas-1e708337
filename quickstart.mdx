Esta guía te lleva desde cero hasta tener la aplicación corriendo y lista para registrar estudiantes, asignar membresías y crear clases.

## Requisitos previos

<CardGroup cols={2}>
  <Card title="Java JDK 11 o superior" icon="coffee">
    Descarga e instala el JDK desde [adoptium.net](https://adoptium.net). Verifica la instalación con `java -version`.
  </Card>
  <Card title="Repositorio del proyecto" icon="git-branch">
    Clona el repositorio o descomprime el archivo fuente. El directorio raíz debe contener la carpeta `src/` y el directorio `datos/`.
  </Card>
</CardGroup>

<Warning>
  El directorio `datos/` debe existir en el directorio de trabajo **antes** de arrancar la aplicación. `BootClass` lee todos los archivos serializados desde ahí al iniciar. Si no existe, la carga fallará con `NullPointerException` al iterar `directorio.listFiles()`.
</Warning>

## Compilar y ejecutar

<Steps>
  <Step title="Crea el directorio datos/ si no existe">
    ```bash
    mkdir -p datos
    ```

    Dentro de `datos/` deben vivir todos los archivos de persistencia (`.usr`, `.est`, `.doce`, `.res`, `.grp`) y el archivo de configuración de la academia `NousAcademy.txt`.
  </Step>

  <Step title="Crea el archivo de configuración de la academia">
    `BootClass` lee `datos/NousAcademy.txt` al arrancar para construir el `AcademiaController`. El archivo debe tener exactamente cinco líneas, una por campo:

    ```text
    900123456-7
    NOUS ACADEMY
    3013772580
    contacto@nousacademy.co
    50000000.0
    ```

    Los campos son, en orden: NIT, nombre, teléfono, correo y capital inicial.
  </Step>

  <Step title="Compila el proyecto">
    Desde el directorio raíz del proyecto, compila todas las clases Java:

    <Tabs>
      <Tab title="javac manual">
        ```bash
        find src -name "*.java" > fuentes.txt
        javac -d out @fuentes.txt
        ```
      </Tab>
      <Tab title="IDE (Eclipse / IntelliJ)">
        Importa el proyecto como **Java Project** y usa el comando *Build Project* del IDE. La clase principal es `com.nousacademy.interfacegrafica.NousSwingApp`.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Ejecuta la aplicación">
    Lanza la clase principal. El método `main` de `NousSwingApp` se ejecuta en el hilo de Swing mediante `SwingUtilities.invokeLater`, instala el tema visual y carga todos los datos antes de mostrar la pantalla de login.

    ```bash
    java -cp out com.nousacademy.interfacegrafica.NousSwingApp
    ```

    ```java
    // NousSwingApp.java — punto de entrada
    public static void main(String[] args) {
        SwingUtilities.invokeLater(() -> {
            Theme.install();
            AcademiaController controller;
            controller = ControllerProvider.cargarDesdeBootClass();
            LoginFrame.show(controller);
        });
    }
    ```
  </Step>
</Steps>

## El proceso de arranque (boot)

Antes de mostrar la ventana de login, `ControllerProvider.cargarDesdeBootClass()` ejecuta la siguiente secuencia:

```java
// ControllerProvider.java
public static AcademiaController cargarDesdeBootClass() {
    BootClass boot = new BootClass();
    boot.cargarUsuarios();     // lee archivos *.usr
    boot.cargarEstudiantes();  // lee archivos *.est
    boot.cargarDocentes();     // lee archivos *.doce
    boot.cargarReservas();     // lee archivos *.res
    boot.cargarGrupos();       // lee archivos *.grp
    return boot.getNousAcademy();
}
```

Cada método recorre `datos/`, filtra por extensión y deserializa los objetos usando `ObjectInputStream`. Al terminar, inyecta los arrays resultantes en el `AcademiaController` central mediante sus setters (`setUsuarios`, `setEstudiantes`, etc.).

<Note>
  Si `datos/` está vacío, todos los arrays quedan en tamaño cero y la aplicación arranca igualmente. Solo necesitas que `NousAcademy.txt` exista y tenga las cinco líneas correctas.
</Note>

## Crear el primer usuario administrador

En una instalación nueva no hay ningún usuario en el sistema. Usa la clase `BootDeUsarioAdmin` para generar el primer administrador directamente en `datos/`:

```java
// BootDeUsarioAdmin.java — ejecutar una sola vez
Usuario usuario = new Usuario(
    "Jeronimo", "Duque", "3013772580",
    "jeronimo.duque@gmail.co",
    TipoDocumento.CC, "1034992245",
    "admin", "123"
);
usuario.copiarFicheroUsuario(
    "datos/" + usuario.getTipoDocumento().name().substring(0, 2)
    + usuario.getNumeroDocumento() + ".usr"
);
```

Esto serializa el objeto `Usuario` en `datos/CC1034992245.usr`. La próxima vez que arranque la aplicación, `BootClass.cargarUsuarios()` lo leerá y lo registrará en el controlador.

<Tip>
  Ejecuta `BootDeUsarioAdmin` como clase independiente (`java com.nousacademy.boot.BootDeUsarioAdmin`) antes del primer arranque de `NousSwingApp`. Solo necesitas hacerlo una vez.
</Tip>

## Iniciar sesión

Cuando aparece la ventana de login, ingresa las credenciales del administrador que acabas de crear:

| Campo | Valor de ejemplo |
|-------|------------------|
| Usuario | `admin` |
| Contraseña | `123` |

`LoginFrame` busca la coincidencia recorriendo en orden los arreglos de `Usuario[]`, luego `Estudiante[]` y finalmente `Docente[]`. Según el tipo del objeto encontrado redirige a `AdminFrame`, `StudentFrame` o `TeacherFrame`.

<Warning>
  Un `Estudiante` sin membresía asignada recibe un error `EstudianteSinMembresia` al intentar iniciar sesión. Primero asigna la membresía desde la cuenta de administrador.
</Warning>

## Primeros pasos tras el login

<Steps>
  <Step title="Crea un estudiante">
    Desde `AdminFrame`, ve a la sección de estudiantes y completa el formulario. El controlador valida duplicados por tipo de documento y número, además de unicidad de nombre de usuario, antes de persistir el archivo `.est`:

    ```java
    // AcademiaController.java
    controller.crearEstudiante(
        "Ana", "García", "3001234567",
        "ana.garcia@nousacademy.co",
        TipoDocumento.CC, "1099887766",
        "ana.garcia", "pass123"
    );
    // persiste en datos/CC1099887766.est
    ```
  </Step>

  <Step title="Asigna una membresía">
    Un estudiante sin membresía no puede iniciar sesión. Usa la opción de membresía definida (basada en `TipoMembresia`) o personalizada (precio y clases privadas a medida). Ambas variantes actualizan el capital de la academia en `NousAcademy.txt`:

    ```java
    controller.asignarMembresiaDefinida(
        TipoDocumento.CC, "1099887766",
        TipoMembresia.ESTANDAR,
        LocalDate.now(), 3   // 3 meses
    );
    ```
  </Step>

  <Step title="Crea un grupo (clase)">
    Los grupos agrupan docentes, estudiantes y reservas bajo un mismo código autogenerado (`GRUP001`, `GRUP002`, ...):

    ```java
    controller.crearGrupo(
        "Yoga Matutino",
        "Clases de yoga de lunes a viernes 7:00-8:00 am"
    );
    // persiste en datos/GRUP001.grp
    ```
  </Step>

  <Step title="Crea una reserva de clase grupal">
    Las reservas representan una sesión de clase con hora, aforo y estado. El código se genera automáticamente y el archivo `.res` se escribe en `datos/`:

    ```java
    controller.crearUnaReservaGrupal(
        LocalDateTime.of(2026, 4, 1, 7, 0),
        "Yoga – Sesión 1",
        "Primera sesión del mes",
        "Respiración y postura",
        60f,   // duración en minutos
        3,     // aforo mínimo
        15     // aforo máximo
    );
    ```
  </Step>
</Steps>
